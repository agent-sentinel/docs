This section provides the complete API reference for the Agent Sentinel cloud platform.

## Base URL

**Production**: `https://platform.agentsentinel.dev`

All endpoints are prefixed with:

```text
/api/v1
```

## Authentication

All API requests require an `Authorization` header:

```bash
Authorization: Bearer <token>
```

The platform accepts two authentication methods:

- **API keys** (recommended for SDK) - Generated in the web console
- **JWT tokens** (for web console access) - Automatic via Clerk

<Tip>
For SDK integration, use **API keys**. They're long-lived and designed for programmatic access.
</Tip>

### Get an API key

1. Log in to [console.agentsentinel.dev](https://console.agentsentinel.dev)
2. Navigate to **Settings** â†’ **API Keys**
3. Click **Generate New Key**
4. Copy the key (starts with `as_`)
5. Use in your requests:

```bash
curl -H "Authorization: Bearer as_your_api_key_here" \
  https://platform.agentsentinel.dev/api/v1/runs
```

## Rate limits

API requests are rate-limited to:
- **1000 requests per minute** per API key
- **10,000 requests per day** per organization

Exceeding limits returns `HTTP 429 Too Many Requests`.

## Error responses

All errors return JSON with a consistent structure:

```json
{
  "detail": "Error message here",
  "error_code": "BUDGET_EXCEEDED",
  "status_code": 400
}
```

Common status codes:
- `400` - Bad request (invalid parameters)
- `401` - Unauthorized (invalid/missing API key)
- `403` - Forbidden (insufficient permissions)
- `404` - Not found
- `429` - Too many requests (rate limited)
- `500` - Internal server error
